双面人设：**Geek 的我 × Rock 的我**。不是简单的 day/night mode 换色，而是两套完全不同的视觉语言 + 有仪式感的切换动画，直接回应 HM "design + eng background" 的反馈——让评审在页面本身感受到 design taste，而不是靠文字辩解。

## 1. Landing Page

### 1.1 结构
- 页面本质是一个**双态容器**：同一个 landing page，根据当前状态（Geek / Rock）整体切换视觉语言、内容板块。
- 默认状态：**Geek 面**（面向 AI/工程岗位场景，第一眼展示 engineering + AI hacker 身份）。
- 切换入口：页面上需要有一个明显的、有仪式感的切换控件（不是右上角小图标式的 day/night toggle），具体形式待设计（见 1.4）。

### 1.2 Geek 面（默认）
- 视觉语言：terminal / 系统感。等宽字体、深色背景、冷色调（青/蓝/绿系）、network/agent 流程图式的装饰动效。
- 内容：AI 项目板块入口 —— WAND、Seen It (youtube-copilot)、**PaperBook** 为主打，dispatch-cli、interview-coach 为佐证卡片。
- 基调：精密、克制、"AI hacker + systems + design taste"。

### 1.3 Rock 面
- 视觉语言：舞台感。暖色调、聚光灯质感、有机曲线、音浪波形动效、手写感/粗体字体。
- 内容：鼓手演出视频 + StageV1 app。内容量刻意少而精，不做陪衬感的堆砌。
- 基调：表现力、审美直觉的佐证，证明 design taste 是长期习惯而非临时包装。

### 1.4 切换交互（本次重点讨论）
- **要求：有仪式感的过渡动画**，且切换效果要**足够明显**，不能被误认为是普通的 day/night mode toggle。
- **已定方向：机械感 + "log off work" 隐喻**
  - 质感基调：机械装置感（不是柔和渐变/淡入淡出的场景过渡，而是像一个物理动作被触发——按钮按下、系统响应、状态切换）。
  - 核心隐喻：从 Geek 面切到 Rock 面，就像**下班 log off**——结束一个"工作会话"，退出系统身份；反向切回 Geek 面则像重新登录/开机进入工作状态。
  - 触发方式：**点击一个按钮**触发整个切换（不是拖动/长按等需要持续动作的交互），点击本身是"一次性的、明确的动作"，呼应 log off 的干脆感。
- 分镜草案（待细化打磨）：
  1. 用户点击按钮（按钮本身可以设计成有物理反馈感的形态，例如拟物化的开关/终端里的一条命令/一个电源键，具体形态待定）。
  2. 按钮触发后，当前面（如 Geek 面）以类似"系统关闭/退出登录"的方式收场：例如界面元素逐一收起、终端式的关闭日志滚动、屏幕短暂黑屏或扫描线效果。
  3. 短暂的"切换中"状态（可选：loading/切换态的过场，强化机械感，比如齿轮/进度条/系统重启提示）。
  4. 新的一面（如 Rock 面）以对应的"开机/登入"方式登场：视觉语言、配色、内容随之切换。
  5. 反向切换（Rock → Geek）复用同一套隐喻，方向对调。
- **已定：按钮沿用常见 day/night mode toggle 的形态**（一个滑动式开关/拨钮），但触发后的过渡效果要做出"log off/log in"的机械感和明显的场景切换，而不是简单的颜色渐变——即：**控件本身克制、熟悉，切换动画本身要有存在感**。
- **音效：暂不加**。
- 待决定项：
  - [ ] "log off / log in" 过场的具体视觉细节（收起动效、扫描线、进度提示等如何呈现）
  - [ ] 过渡总时长（需要足够有仪式感，但不能拖慢体验——待定具体秒数）
  - [ ] 文案是否配合出现（例如短暂显示 "logging off..." / "logging in..." 之类的系统提示文案）

## 2. 项目展示（Geek 面）

### 2.0 项目清单更新
新增第 5 个 AI 项目：**PaperBook**（DeepMind hackathon 作品）
- 路径：`/Users/lfy_macbook_air/Documents/GitHub/gemini_hackathon`
- 一句话：把科研论文 PDF 端到端转化为 ~90 秒科普动画视频。核心创新是"类比世界"（把论文映射到一个结构相似的日常场景，如"餐厅厨房"）作为解释框架，再配合"命名时刻协议"（先类比、后专业术语）降低理解门槛。技术上是 11 步 DAG pipeline（doc_ir → sketch → world → glossary → claim → visual_assets → storyboard → Veo 3.1 生成 → ffmpeg 合并），有并发 clip 生成 + 实时进度写入的工程设计。
- 定位：和 WAND、Seen It 一样属于主打项目——但 PaperBook 更偏"信息转译/叙事设计"（把复杂内容变得可理解），补充了另一种 design taste 的证明角度（不止交互设计，还有内容/叙事设计）。

Geek 面主打项目更新为：**WAND、Seen It、PaperBook** 三个（原有的双主打改为三主打），dispatch-cli、interview-coach 仍为佐证卡片。

### 2.1 核心设计理念：每个项目都是「可玩的」interactive demo
不做静态截图 + 文字描述的传统 portfolio 卡片。**每个项目在页面内还原一个简化版的产品 UI，观众可以直接动手操作，触发一段模拟的产品体验**，用"玩一遍产品"代替"看一遍介绍"——这本身就是最直接的 design taste 展示方式。

### 2.2 各项目 interactive demo 设计

**PaperBook**（已定基本玩法，作为其他项目的设计范式参考）
- 页面内还原一个简化版 PaperBook 产品 UI。
- 旁边放一个可拖拽的 PDF 小图标。
- 用户把 PDF 图标拖进 UI 的上传区域 → 触发一段"处理中"的读条/进度动效（呼应真实产品里 11 步 pipeline 的"处理感"，可以做成几个阶段性文字提示滚动，比如 doc_ir → world mapping → storyboard → rendering，增强真实感）。
- 处理完成后，播放一段预先准备好的生成动画（真实产出的科普视频片段或代表性 demo 片段）。
- 交互闭环：拖拽 → 处理动效 → 结果播放，全程不跳出当前页面。

**WAND**（已定方向）
- 核心卖点是"免打字免鼠标、看得见听得懂能动手"。
- 玩法：还原一个简化的"屏幕 + 摄像头取景"界面，配一个麦克风按钮。用户点击几个预设的"说话"选项（贴合 README 场景，如做饭/购物时的指令，例如"帮我点开这个最便宜的"）。
- 交互流程：点击预设指令 → 出现语音转文字气泡（体现"听得懂"）→ 画面中指针/高亮框自动移动到对应 UI 元素并完成点击（体现"看得见、能动手"）。全程不出现打字/鼠标操作，直给"免打字免鼠标"这一核心卖点。
- 待实施细化：具体 UI 视觉细节、指针移动动效实现方式、示例指令文案。

**Seen It (youtube-copilot)**（已定方向）
- 核心卖点是"in-video search + voice control，不打断观影体验"。
- 玩法：还原一个迷你视频播放器（素材用 README 举例的场景，如"拧螺丝"教程片段），用户点几个预设的自然语言查询 chip（如"跳到他拧螺丝的地方"），播放进度条自动跳转到对应片段并高亮时间戳（体现精准定位）。
- 关键细节：跳转完成后，右下角安静滑入一张"推荐商品"小卡片，几秒后自动收起——这个细节体现"推荐但不打断体验"的克制感，是本项目最重要的 design taste 佐证点，实现时不能省略。
- 待实施细化：具体 UI 视觉细节、跳转动效实现方式、示例视频素材与查询文案。

**dispatch-cli**（已升级为独立彩蛋板块，见 2.3）

**interview-coach**（佐证卡片，暂不做完整 interactive demo）
- 定位是佐证卡片，暂定用更轻量的展示形式（如静态截图 + 关键说明），是否做成小型 interactive demo 待后续讨论，视整体页面节奏和开发工作量决定。

### 2.3 dispatch-cli 彩蛋板块（已定方向）
不再作为普通佐证卡片，升级为一个**独立设计的幽默彩蛋板块**，用自黑+巧思本身证明 design taste（懂得什么时候克制、什么时候玩）。

**叙事框架**：前面展示了这么多 creative 项目，哪来的时间做？——引出"我最丑但最好用的工具"：dispatch-cli（远程指挥家里电脑干活的工具）。

**板块内容**：
- 标题类似："这么多创意项目哪来的时间？—— 我最丑但最好用的工具"
- 几个功能按钮，点击触发模拟动画（类似终端刷日志/commit 滚动效果），体现"远程指挥家里电脑"的真实使用场景：
  - 「给 PaperBook 加个新 feature」
  - 「继续肝 WAND」
  - 「修 Seen It 的 bug」
- **彩蛋按钮**：「启动扫地机器人」，配文案："别点，你真的会连到我家扫地机器人"
  - 点击后：一个扫地机器人图标从页面一角出现，开始"打扫"，把页面上的内容元素逐个扫走/清空（动画形式待实施细化，可以是元素逐个被"扫"出视口或淡出）
  - 最终清空到只剩下 day/night（Geek/Rock）切换按钮，配一句俏皮收尾文案（如："好，扫干净了，去试试切换 Night 模式吧"）
  - 功能性双重作用：既是幽默彩蛋，也是把用户自然引导去发现 Geek/Rock 切换入口的巧妙设计。
- 待实施细化：扫地机器人清空动画的具体实现方式、各按钮触发动画的具体视觉效果、文案措辞打磨。

## 5. 技术实现与部署

### 5.1 技术栈（已定）
- **Next.js + TypeScript**，static export 部署到 GitHub Pages。
- 理由：
  - 每个作品的 interactive demo 天然是独立组件（PaperBook 拖拽处理、WAND 指令模拟、Seen It 播放器跳转、StageV1 滑卡、dispatch-cli 彩蛋按钮），组件化便于单独开发和调试。
  - TypeScript 对"很多小交互状态机"的项目能减少低级 bug。
  - Next.js static export 对 GitHub Pages 友好，且不影响后续引入动效库（如 Framer Motion）实现 1.4 中"仪式感转场"效果。
